# 图片资源说明

## 📸 图片要求

### 规格建议
- **格式**：JPG（推荐）或 PNG
- **尺寸**：宽度 900px - 1200px（高度自适应，保持3:4比例）
- **大小**：单张不超过 500KB（使用 TinyPNG 压缩）
- **分辨率**：72-150 DPI（屏幕显示足够）

### 文件命名规范
```
pages/
├── cover.jpg          # 封面（硬页）
├── page-1.jpg         # 第1页内容
├── page-2.jpg         # 第2页内容
├── page-3.jpg         # 第3页内容
├── ...                # 更多页面
└── back.jpg           # 封底（硬页）
```

---

## 🛠️ 快速生成图片

### 方法一：从PDF导出
1. 打开 PDF 文件
2. 使用 Adobe Acrobat：文件 → 导出到 → 图像 → JPEG
3. 或使用在线工具：https://smallpdf.com/cn/pdf-to-jpg
4. 将生成的图片按顺序重命名

### 方法二：从PPT导出
1. 在 PowerPoint 中完成设计
2. 文件 → 另存为 → 选择 JPEG 格式
3. 选择"所有幻灯片"
4. 重命名为 `page-1.jpg`, `page-2.jpg`...

### 方法三：从Word导出
1. Word → 另存为 PDF
2. 再将 PDF 转为图片（见方法一）

### 方法四：截图方式
1. 打开原始画册或设计稿
2. 使用截图工具逐页截取
3. 保存为 JPG 格式并重命名

## 🎨 图片优化技巧

### 压缩工具
- **TinyPNG**：https://tinypng.com/（最佳，无损压缩）
- **Squoosh**：https://squoosh.app/（Google出品，可调节质量）
- **ImageOptim**：Mac专用批量压缩工具

### 压缩建议
- 照片类：质量 70-80%
- 文字类：质量 80-90%
- 混合类：质量 75-85%

### 批量处理
使用 Photoshop 动作或在线批量压缩工具

---

## ⚠️ 注意事项

1. **文件名必须准确**：区分大小写，确保与HTML中引用一致
2. **图片方向正确**：横向画册用横向图片，纵向画册用纵向图片
3. **保持一致性**：所有图片尺寸比例应相同
4. **测试加载速度**：在慢速网络下测试，确保流畅浏览
5. **备用方案**：如果某些页没有图片，可以只保留文字内容

---

## 🔍 故障排查

### 图片不显示
- 检查文件名是否正确（包括扩展名）
- 检查路径是否正确（`pages/page-1.jpg`）
- 检查浏览器控制台是否有404错误
- 清除浏览器缓存后刷新

### 图片变形
- 确保所有图片宽高比一致
- 调整 CSS 中的 `object-fit` 属性

### 加载缓慢
- 进一步压缩图片
- 考虑使用 WebP 格式（需修改HTML引用）
- 启用 CDN 加速

## 📊 示例图片结构

如果您有32页的画册：
```
pages/
├── cover.jpg          # 封面
├── page-1.jpg         # 第1页
├── page-2.jpg         # 第2页
├── page-3.jpg         # 第3页
├── page-4.jpg         # 第4页
├── ...
├── page-30.jpg        # 第30页
├── page-31.jpg        # 第31页
└── back.jpg           # 封底
```

在 `index.html` 中添加对应的 `.page` div 块即可。
